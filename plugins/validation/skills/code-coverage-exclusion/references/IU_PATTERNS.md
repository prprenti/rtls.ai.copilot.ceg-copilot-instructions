# IU-Specific Coverage Exclusion Patterns

## D2D FDI Module

### Pattern-Based Exclusions

| Pattern | Description | Examples |
|---------|-------------|----------|
| `.*fscan.*` | D2D functional scan | `d2d_hde_fscan_mode`, `fscan_byprst_b`, `fscan_latchopen` |
| `.*ijtag.*` | D2D IJTAG signals | `d2d_hde_ijtag_tck`, `ijtag_select`, `ijtag_capture` |
| `^fp_.*` | Floorplan configs | `fp_cfg_portid`, `fp_cfg_ready_portid` |

### DFT Signals to Exclude

```
fscan_mode, fscan_shiften, fscan_clkungate, fscan_rstbypen
fscan_latchopen, fscan_latchclosed_b, fscan_byprst_b, fscan_byppst_b
fscan_clkungate_syn, fscan_clkgenctrl
```

### IJTAG Signals to Exclude

```
<instance>_ijtag_tck, <instance>_ijtag_tdi, <instance>_ijtag_tdo
<instance>_ijtag_si, <instance>_ijtag_so
<instance>_ijtag_shift, <instance>_ijtag_capture, <instance>_ijtag_update
<instance>_ijtag_select, <instance>_ijtag_reset_b
```

### Common D2D Strap Patterns

```systemverilog
.d2d_hde_fscan_mode(0),                    // Tied to 0
.d2d_hde_fscan_clkungate(0),               // Tied to 0
.d2d_hde_fscan_byprst_b(1),                // Tied to 1 (active low)
.fp_cfg_portid(d2d_common_pkg::DEV_CTRL_DEST_PORTID),
.fp_cfg_ready_portid(16'h0081),
.d2d_hde_LocalSegmentID('0),
```

## Adding New IU Patterns

To add patterns for a new IU:

1. Identify IU-specific DFT/test signal naming conventions
2. Document regex patterns with examples
3. List common strap connections from TB
4. Add section to this file following D2D format
